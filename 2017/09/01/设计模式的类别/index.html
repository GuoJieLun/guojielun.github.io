设计模式的类别

2017-09-01 设计模式 0 评论 字数统计: 298 字 阅读时长: 1 分钟

加粗的字体是常见的设计模式

创建型设计模式

描述：创建型设计模式专注于处理对象机制,已适合给定情况的方式来创建对象的几本方法可能导致项目的复杂性添加,而这些模式只在通过创建过程来解决这种问题。

  • 模式包括:
    • Constructor(控制器)
    • Abstract(抽象)
    • Factory(工厂)
    • Prototype(原形)
    • Singleton(单例)
    • Builder(生成器)

结构行设计模式

描述:结构型模式于对象组合有关,通常可以用于找出在不同对象之间建立关系的简单方法。这种模式有助于确保在系统某一部分发生变化时，系统的整个结构不需要同时改变。同时对于不适合因某一特定目的改变的系统部分，这种部分也能够帮助它们重组。

  • 模式包括：
    • Decorator(装饰者)
    • Facade(外观)
    • Adapter(适配器)
    • Proxy(代理)

行为设计模式

描述：行为模式专注于改善或简化系统找那个不同对象的通信

  • 模式包括：
    • lterator(迭代器)
    • Madiator(中介者)
    • Observer(观察者)
    • Visitor(访问者)
